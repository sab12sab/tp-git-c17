# Comprendre le Version Control

## Qu’est-ce que le Version Control ?

Le Version Control (ou contrôle de version) est un système qui permet de suivre les modifications apportées à des fichiers au fil du temps. Il permet de conserver un historique complet des changements, de revenir à une version précédente et de travailler en collaboration sur un même projet sans écraser le travail des autres.

## Pourquoi en a-t-on besoin ?

Le contrôle de version est essentiel en développement logiciel pour plusieurs raisons :

- Suivre l’historique des modifications
- Revenir à une version stable en cas d’erreur
- Travailler en équipe sur le même projet
- Gérer différentes versions du code (branches)
- Assurer la traçabilité et la sécurité du projet

## Outil utilisé

Git est un système de contrôle de version distribué très utilisé. Il permet de gérer les versions localement et de synchroniser le projet avec des plateformes comme GitHub.
